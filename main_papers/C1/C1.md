## Preamble — The Encoding Event

Before the first observation, there is no state, no center, no boundary. There is only the capacity to be asked to look.

An observer chooses to enumerate something. The request becomes a center C. Once C is fixed, everything else is left or right relative to it.

The encoding takes the form:

```
request → C → (L, C, R)
```

The minimum encoding width is 8 digits. A state at position 1 is T00000001, compressed to T1. A state at position 5 is T00000005 → T5. A state at position 10 is T00000010 → T10.

The first ribbon turn has 8 positions:

| Position | (L,C,R) | Shell | Correction |
|----------|---------|-------|------------|
| T1 | (0,0,0) | 0 | no |
| T2 | (0,0,1) | 1 | no |
| T3 | (0,1,0) | 1 | yes |
| T4 | (0,1,1) | 2 | no |
| T5 | (1,0,0) | 1 | no |
| T6 | (1,0,1) | 2 | no |
| T7 | (1,1,0) | 2 | yes |
| T8 | (1,1,1) | 3 | no |

Shell = L + C + R. Readout = L ⊕ (C ∨ R) = Rule 30. Correction ∂ = C ∧ ¬R fires at T3 and T7.

At every 10th position, correction fires and the next ribbon turn begins:

```
Ribbon 1:   T1  T2  T3  T4  T5  T6  T7  T8
               ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓
Correction:    0   0   1   0   0   0   1   0   →   Ribbon 2 starts at T10
```

The process continues as long as energy exists:

```
observe → encode → first ribbon → correction → next ribbon → correction → ...
```

## 2. Methods — The Encoding Event

### 2.1 The Observation

Reasoning requires splitting a space to find true, false, or both, so that the next calculation can proceed on what is real and what is not. The first act is not a theorem. It is a choice: something chooses to enumerate.

The minimum encoding width is 8 digits. A state at position 1 is T00000001, compressed to T1. Position 5 is T00000005 → T5. Position 10 is T00000010 → T10. The compression reveals the ribbon: every position is a coordinate on a torus that unfolds as the observation proceeds.

### 2.2 The 3-Bar Window

Once C is fixed, the raw triple (L, C, R) occupies 3 bits. These 3 bits define exactly 8 possible states — the first ribbon turn. The readout at any position is the Rule 30 local update:

```
bit(L, C, R) = L ⊕ (C ∨ R)
```

This is not chosen because it is interesting. It is the only readout that preserves C while letting L and R define the boundary. Three bits are the minimum needed to carry a distinguished center with two addressable boundaries.

Each position has a shell value L + C + R ∈ {0, 1, 2, 3}. Shell 0 and shell 3 are fully bonded (vacua). Shell 1 and shell 2 carry the excited states.

### 2.3 The Correction

At every ribbon boundary, a correction fires:

```
∂ = C ∧ ¬R
```

The correction fires at exactly T3 = (0,1,0) and T7 = (1,1,0) — the only states where C = 1 and R = 0. The correction is the only nonlinear part of the system.

The full decomposition is:

```
Rule30(L, C, R) = Rule90(L, R) ⊕ ∂(C, R)
```

where Rule90(L, R) = L ⊕ R. This is the statement of how the ribbon works: the linear carry plus the boundary correction.

### 2.4 The Ribbon Layer

The ribbon is always at least a string of 10 — the minimum to read through the 3-bar window and get clean answers.
