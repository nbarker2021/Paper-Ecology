# FLCR-01 Workbook - Grounding Contract And Axiom Discipline

## Purpose

This workbook turns the formal paper into a reproducible inspection exercise.
It is not a separate proof. It is the analog/digital bridge that lets a reader
reconstruct the object, claim lane, evidence route, and remaining obligation.

The workbook is also the seed of the analog toolkit. Its main design target is
universal STEM access: a person should eventually be able to use the kit to
reason about the same structure without electricity, internet, computers, or
paper. The workbook may use files and code during publication, but each
exercise should preserve a lower-tech equivalent whenever possible.

## Completion Criteria

- The reader can identify the main object without importing downstream claims.
- Every strong statement has a claim lane.
- Every missing datum is an obligation, not an unlabeled gap.
- Any CAM/crystal query is linked to a source path or marked as pending.

## Guardrail Interpretation

Use the guardrails as automation controls. They are checks on AI agents,
validators, build scripts, generated prose, and claim-promotion workflows. They
are not a ban on speculative thinking. When a grand claim is not yet fully
proved, write the largest intended claim, list the proved pieces, and route the
remaining distance as open obligations.

| Item | Workbook treatment |
|---|---|
| Grand claim | State it clearly as the intended direction. |
| Proven part | Attach receipt, theorem, finite enumeration, standard formalism, CAM route, or calibration row. |
| Open obligation | Treat it as the next channel of work from the declared prestate. |
| AI guardrail | Prevent silent promotion across lanes. |
| Human hypothesis | Preserve it as a research lead with evidence requirements. |

## Label-Preserving Analog Transfer

The workbook must demonstrate that the author's interpretation changes labels,
coordinates, access paths, or analog handles, not the underlying addressed
state. The analog route should therefore be auditable against the formal claim
lane and the original ribbon role.

## State-Preserving Analog Contract

Analog invariance test: a sealed ledger or completed window whose contents are preserved while the next layer receives only typed imports.

The workbook exercise is to fill this table with concrete receipts, equations,
citations, code paths, or CAM/crystal queries:

| Check | Required content |
|---|---|
| Received state | Exact source object entering the paper |
| Formal carrier | Accepted theorem, construction, verifier, or receipt being used |
| Interpretation label | The author's label, analogy, or access-path change |
| Invariant state | What remains unchanged under that relabeling |
| Produced state | What downstream paper is allowed to import |
